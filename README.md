📱 Potok - Социальная платформа для микроблогов
Potok — это современная социальная платформа для создания микроблогов, обмена сообщениями и общения в сообществах. Проект представляет собой полнофункциональный аналог популярных социальных сетей, разработанный с использованием Django и предоставляющий пользователям широкие возможности для коммуникации и самовыражения.

✨ Ключевые особенности

👥 Профили и подписки

Расширенные профили пользователей с аватарами и личной информацией.

Система подписок на других пользователей для формирования персональной ленты.

Создание, редактирование и удаление текстовых постов.

Древовидные комментарии для обсуждения постов.

Отправка и получение текстовых сообщений в реальном времени.

Список активных чатов и история переписки.

Отметка о прочтении сообщений.

👪 Сообщества

Создание и управление тематическими сообществами (модуль communities).

Возможность вступать в сообщества и публиковать посты от их имени.

Ролевая модель внутри сообщества (создатель, администратор, участник).

Отдельная лента с постами сообщества.

🛡️ Модерация и безопасность

Выделенное приложение moderation для управления контентом и пользователями.

Система жалоб на посты, комментарии и пользователей.

Возможность блокировки пользователей модераторами.

Логирование действий для аудита (папка logs).

🏗️ Архитектура системы

Технологический стек

Backend: Django 4.x, Python 3.10+

База данных: SQLite (для разработки), с возможностью миграции на PostgreSQL

Фронтенд: HTML5, CSS3, JavaScript

Ключевые библиотеки и модули:

django.contrib.auth — аутентификация

Pillow — работа с изображениями

python-dotenv — управление переменными окружения

Основные модули приложения (Django apps)

accounts — управление пользователями, регистрация, профили, подписки.

posts — основная логика постов, комментариев, лайков и репостов.

communities — функционал сообществ.

messenger — система личных сообщений.

moderation — инструменты для модераторов и администраторов.

🚀 Установка и запуск

Требования

Python 3.10 или выше

Git

Virtualenv (рекомендуется)

Шаг 1: Клонирование репозитория

bash
git clone https://github.com/minato131/potok.git
cd potok
Шаг 2: Настройка виртуального окружения

bash
# Создание виртуального окружения
python -m venv .venv

# Активация (Linux/Mac)
source .venv/bin/activate

# Активация (Windows)
.venv\Scripts\activate
Шаг 3: Установка зависимостей

bash
pip install -r requirements.txt
Шаг 4: Настройка переменных окружения
Создайте файл .env в корне проекта на основе примера:

env
# Django settings
SECRET_KEY=ваш_секретный_ключ_django
DEBUG=True
DJANGO_ALLOWED_HOSTS=localhost 127.0.0.1

# Database
DATABASE_URL=sqlite:///db.sqlite3

# Другие настройки
# ...
Шаг 5: Выполнение миграций базы данных

bash
python manage.py migrate
Шаг 6: Создание суперпользователя (администратора)

bash
python manage.py createsuperuser
# Следуйте инструкциям в терминале
Шаг 7: Запуск сервера разработки

bash
python manage.py runserver
Шаг 8: Доступ к приложению
Откройте браузер и перейдите по адресу: http://127.0.0.1:8000/
Для доступа к админ-панели Django: http://127.0.0.1:8000/admin

🌐 API Endpoints (Основные)

Проект построен на стандартных Django-представлениях. Ниже приведены основные URL-маршруты:

 (пустой путь) — главная страница (лента постов).

accounts/ — пространство имен для управления пользователями (логин, логаут, регистрация).

messenger/ — основное приложение для личных сообщений.

moderation/ — основное приложение для модерации.

🧪 Тестирование

Для запуска тестов используйте стандартную команду Django:

bash
# Запуск всех тестов
python manage.py test

# Запуск тестов для конкретного приложения
python manage.py test accounts
python manage.py test posts
python manage.py test communities

# Запуск тестов с подробным выводом
python manage.py test -v 2
Типы тестов (Рекомендуется к реализации)

FUNC-01: Функциональные тесты ленты постов и взаимодействий.

🔧 Устранение неполадок

Ошибки импорта модулей

bash
# Убедитесь, что виртуальное окружение активировано
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows

# Переустановите зависимости
pip install -r requirements.txt --force-reinstall
Проблемы с базой данных

bash
# Удаление старой базы данных и создание новой
rm -f db.sqlite3
python manage.py migrate

# Создание суперпользователя заново
python manage.py createsuperuser
Ошибки запуска сервера

Убедитесь, что порт 8000 не занят другой программой.

Проверьте, что все миграции применены (python manage.py migrate).

Проверьте значения DEBUG и ALLOWED_HOSTS в файле potok/settings.py или .env.
